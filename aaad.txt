READ ME FIRST
─────────────
The whole Elif-Ba module lives in a single React + TypeScript file:
  src/app/components/ElifBaPage.tsx

Stack the outside system must produce code for:
  • TypeScript (strict — no `any`; keep the file's existing style).
  • React 18 function components with hooks.
  • Tailwind CSS utility classes (the app is Tailwind-only, no CSS modules,
    no styled-components, no CSS-in-JS libs).
  • Arabic glyphs come from the CSS constant ARABIC_FONT (Amiri + Scheherazade
    New + serif fallback). Reuse it verbatim for any Arabic text.
  • Audio helper: `useAudio()` + `audioPath(letterId, harakatId?)`. Do not
    invent new audio filenames — the mp3s live in /public/audio/.
  • Progress + i18n plumbing (setProgress, tr(), lang: 'nl' | 'tr') is already
    wired into the parent StageView. New games should follow the existing
    prop shape:
        function XyzGame({ letters, onComplete, lang }: {
          letters: ArabicLetter[]; onComplete: (stars: number) => void;
          lang: 'nl' | 'tr';
        }) { ... }
    and call onComplete(0..3) exactly once when the round ends.
  • Failure rule: losing all 3 lives MUST call onComplete(0), never a
    consolation star. StageView renders the retry screen from that.
  • Every quiz-style game must cap its queue at ≤ 15 questions.

Illustrations / backgrounds
  • Deliver as inline SVG strings or standalone .svg files in /public/img/.
  • Keep them lightweight (<40 KB each, no external hrefs — CSP blocks them).
  • Provide both a "light-sky" and a "night" variant when it helps the mood.
  • Palette should stay in the current Tailwind blue/emerald/amber/violet
    range so section gradients still fit around them.

If your system produces graphics separately, it can also just hand back
PNG/WebP assets — but SVG scales best on tablets and phones.

1. LEARN — "Kijk en luister"
   (LearnGame, HarakatLearnGame, SignLearnGame, FormLearnGame)
═══════════════════════════════════════════════════════════════════════════════

Current state
  A single big white card in the middle with the Arabic glyph and a "🔊"
  speaker chip, a name label below, prev/next buttons. Functional but flat.

What we want
  • A calm, storybook feel: soft parchment/paper card centred on a scene
    that hints at where Arabic letters "live" (open Quran on a wooden desk,
    lantern in a night scene, garden with palm trees, etc).
  • Subtle idle animation: the card should "breathe" (2s ease-in-out) and
    the speaker chip should get a small pulse ring while audio plays.
  • Bigger tap target for the glyph itself (already 208×208; keep or grow).
  • Peltek letters need an unmistakable visual accent — a soft amber halo
    behind the card and a friendly tongue-between-teeth icon.

Prompt to hand to the design/illustration system
  "Design a warm, storybook 'letter of the day' scene for a children's
  Arabic-alphabet app. The centrepiece is an 8:9 rounded paper card that
  will hold one large Arabic letter (rendered from Amiri font on the client
  side — leave the card blank). Around the card, add a subtle scene: open
  Quran with turning pages, a hanging lantern, palm-frond corners, or a
  small carpet. Ages 4–8, calm palette (emerald, sky-blue, cream, warm
  amber accents), no photorealism, no faces. Provide light and night
  variants. Output as one self-contained SVG per variant, no external
  images, viewBox 0 0 800 900, max 40 KB, no gradients that break in
  Safari 15."

Code hooks the system should be aware of
  • The parent stage's background gradient is `bg-gradient-to-b from-{color}`;
    the scene must sit on transparency so it stacks correctly.
  • Reuse SPEAKER_BADGE for the audio chip so the placement stays consistent.

2. LISTEN & PICK / NAME MATCH — "Luister & Kies" and "Naam Quiz"
   (ListenPickGame, NameMatchGame)
═══════════════════════════════════════════════════════════════════════════════

Current state
  Simple grid of four white cards. Feedback: correct→green, wrong→red.

What we want
  • A "classroom rug" scene: cards feel like picture blocks laid out on
    a woven rug, with a friendly teacher-owl mascot in a corner reacting
    to the answer (happy on correct, gently surprised on wrong).
  • Cards should have a small tilt and shadow so they feel physical.
  • Confetti already exists; a scene reaction is enough.

Prompt
  "Illustrate a friendly classroom rug scene as a background for a 2×2
  card grid used in an Arabic-alphabet quiz for children ages 4–8.
  Include a small owl or camel mascot in the bottom-right that has three
  poses: idle, happy (arms up, sparkle), and encouraging (paw up with a
  gentle 🤔). Do not draw the cards themselves — leave the centre of the
  scene empty so the React grid overlays cleanly. Palette: warm rug reds,
  cream, emerald, amber. Provide the mascot as three separate SVGs so the
  code can swap them on state change. viewBox 0 0 1000 700."

Code hooks
  • Mascot state maps to component state `feedback: 'correct' | 'wrong' | null`.
  • Please deliver mascot poses as ~120×120 SVGs so they can drop in with:
      `<Mascot pose={feedback === 'correct' ? 'happy' : feedback === 'wrong' ? 'encourage' : 'idle'} />`

3. BALLOON POP — letters and harakat variants
   (BalloonPopGame, HarakatBalloonPopGame)
═══════════════════════════════════════════════════════════════════════════════

Current state
  Sky-blue gradient background, three cloud emoji, sun emoji, grass strip
  at the bottom. Balloons rise from the grass. Works well but is
  emoji-only art.

What we want
  • A proper hand-drawn sky scene: rolling hills at the bottom, a few
    tufts of grass and flowers on the horizon, layered clouds that drift
    slowly across the sky (CSS `@keyframes` translateX, 30s+ loop).
  • The sun should have a soft rays halo animating gently (rotate slow).
  • Bird silhouettes crossing occasionally are a nice touch (once every
    10–15s, right-to-left).
  • The balloons themselves should keep their current React DOM (they
    carry Arabic glyphs and need the physics loop) but their gloss and
    string should be replaced with an SVG so they look painted, not flat.

Prompt
  "Design a serene sky-and-hills background for a children's balloon-pop
  Arabic-alphabet game (ages 4–8). Layers, back-to-front: gradient sky
  (sky-blue → cream), soft cumulus clouds (2 sizes, will loop
  horizontally in CSS), a warm sun with a soft halo (will rotate slowly),
  rolling green hills at the bottom taking ~15% of viewport height, with
  small flowers and tufts. NO balloons, NO letters — those overlay from
  React. Provide as a single 1600×1000 SVG plus two extra assets: one bird
  silhouette (used for crossing animation) and one balloon SVG template
  (rounded top, string, highlight) that will be tinted per instance via
  `fill: currentColor`. Palette should match Tailwind sky-300/emerald-500.
  Max 60 KB combined."

Code hooks
  • Replace the emoji sun/clouds/grass in BalloonPopGame with the new
    background SVG (as `<img>` or inline SVG, absolute-positioned).
  • The balloon template SVG becomes the balloon body; the code overrides
    `color` per instance from BALLOON_COLORS.
  • Harakat variant: same visuals, different content — no new art needed.

4. FALLING LETTERS — "Vangen"
   (FallingLettersGame)
═══════════════════════════════════════════════════════════════════════════════

Current state
  Sky-to-grass gradient with a 🧺 basket emoji and floating letter cards.
  Works but the basket is a wooden rectangle with a basket emoji inside.

What we want
  • A picnic-meadow feel: same sky as balloon but with a picnic blanket
    peeking at the bottom-left, a tree on one side, and butterflies
    fluttering.
  • A proper woven basket illustration (SVG) replacing the emoji, with a
    slight bob animation when it moves.
  • Letter cards should look like Scrabble tiles (cream tile, beveled
    edges, wood-brown border) instead of plain white rectangles.

Prompt
  "Design a picnic-meadow background and two supporting props for a
  children's Arabic-alphabet 'catch the letter' game (ages 4–8). Assets:
  (a) background SVG 1600×1000 — same sky/hills family as the balloon
      scene, plus a small picnic blanket peek and a tree in one corner.
  (b) woven picnic basket SVG ~200×130 with visible handle, viewed
      slightly from above so letters can drop into it visually.
  (c) 'letter tile' SVG ~180×180 — cream tile with wood-brown bevel and
      a subtle inner shadow, empty centre (the Arabic glyph is drawn by
      React on top). Provide as three files. Palette matches balloon
      background. Max 60 KB combined."

Code hooks
  • `handleMove` sets `basketX`; keep that shape. The basket SVG replaces
    the current amber-brown div.
  • Letter cards use `item.letter.arabic` centred on the tile SVG.

5. WHACK-A-MOLE — "Meppen"
   (WhackAMoleGame)
═══════════════════════════════════════════════════════════════════════════════

Current state
  Green radial-gradient field, 6 mole heads with letters at random
  positions, grass emoji tufts, dirt oval holes underneath the moles.
  Positions scatter but the moles are simple brown circles.

What we want
  • A proper hand-drawn meadow with wildflowers, stones, and a wooden
    fence far in the background.
  • Mole illustration with two states: peeking (default) and whacked
    (eyes X'd, small stars). The Arabic letter sits on the mole's belly.
  • Dirt hole with a slight rim and a soft shadow to feel 3D.
  • Optional: a mallet cursor that follows the pointer on desktop.

Prompt
  "Illustrate a whack-a-mole style Arabic-alphabet game background and
  props for children (ages 4–8). Assets:
  (a) background SVG 1600×1000 — meadow with wildflowers, small rocks,
      distant wooden fence, and a soft sun. Muted palette to keep focus
      on the moles. Leave the centre open for 6 scattered holes drawn by
      React.
  (b) hole SVG ~150×90 — dirt oval with rim, soft cast shadow.
  (c) mole SVG ~180×180 — friendly cartoon mole peeking up, empty round
      belly-badge in the centre where the Arabic letter is overlaid. Also
      provide a 'whacked' variant with X eyes and swirling stars.
  (d) optional mallet cursor SVG ~64×64.
  Palette: emerald, warm brown, cream. Max 80 KB combined."

Code hooks
  • `holePositions` array already scatters 6 slots — the hole and mole
    SVGs stack there via absolute positioning.
  • `whacked === i` and `wrongHit === i` map to the mole's 'whacked'
    variant + red flash on wrong hit.

   (MemoryGame)

What we want

    geometric pattern on the back, and a clean cream front.
  • A gentle flip animation on tap (CSS 3D flip, 400ms).

Prompt

  Arabic-alphabet children's game (ages 4–8). Deliver:
  (a) card back SVG 320×420 — geometric Islamic-art pattern in emerald +
      cream, avoiding any Arabic script (the front holds it). Should tile
      cleanly if repeated.
  (b) card front SVG 320×420 — plain cream tile with a thin gold border,
      empty centre (React draws the glyph or name label there).
  (c) 'matched' overlay SVG 320×420 — subtle emerald glow + a checkmark
      badge in the top-right corner.
  Palette: emerald-500, cream-100, gold-400 accent. Max 40 KB combined."

Code hooks
  • Current `flipped` / `matched` state stays the same; the SVGs replace
    the current div backgrounds inside the button.
  • Add `transform-style: preserve-3d` + rotateY on flip (Tailwind
    arbitrary variants).

7. HARAKAT LEARN / QUIZ
   (HarakatLearnGame, HarakatQuizGame)
═══════════════════════════════════════════════════════════════════════════════

Current state
  Learn: chip row at top, big white card with letter+harakat, next button.
  Quiz: speaker button, small letter tile, 3 answer buttons showing
  ◌ + harakat mark.

What we want
  • Both screens should share a "vocal music" theme: soft musical staves
    or wave lines in the background, warm sunset palette to distinguish
    from the letter section.
  • The harakat mark (fatha / kasra / damma) should be colour-coded
    consistently: fatha=amber, kasra=blue, damma=emerald — this already
    exists in HARAKATS. Please use these three as accent colours in the
    illustrations.
  • Quiz answer buttons: the U+25CC placeholder already renders; wrap it
    inside a small "musical note" halo tinted with the harakat colour.

Prompt
  "Design a warm 'vocalisation' background for children's Arabic-alphabet
  harakat games (ages 4–8). Assets:
  (a) background SVG 1600×1000 — soft wavy sound lines drifting upward,
      abstract musical staves, sunset gradient (amber → dusty rose →
      cream). No literal instruments. Leave centre uncluttered.
  (b) three small 'harakat note' badges 96×96 each — a musical note
      shape whose head is tinted amber/blue/emerald respectively for
      fatha / kasra / damma. The dotted-circle-with-mark glyph (rendered
      by React) overlays the note's head.
  Palette: amber-500, blue-500, emerald-500, sunset amber → rose. Max
  60 KB combined."

Code hooks
  • HARAKATS[i].color is the source of truth for the three tints.
  • Learn stage's chip row already uses HARAKATS; the badge SVGs slot in
    behind each chip.
